    Warnings (non-blocking)

      Camera source settings

      Set a camera's RTSP URL and the POS workstation ID that maps to it. Saves go through PATCH /api/v1/admin/cameras/{camera_id}, persist to config.yaml, and are audited. Restart/reload the segment recorder for an RTSP URL change to take effect.

      Prompt editor

      Edits go through PATCH /api/v1/admin/prompts/{camera_id}. The backend refuses unsafe phrasing and writes an audit log entry for every save. The review-safe contract cannot be regressed via this UI.

      Safety scan (live load): —

      Camera ROIs & model views

      Numeric ROI editor with optional visual calibration. Each ROI is a labeled (x, y, w, h) in source-frame pixels. Per-model assignments control how the active pipeline crops or filters evidence: Falcon crops to the union and translates detection boxes back to full frame; SAM 2 and OCR filter detections whose centre falls inside the assigned union; Qwen3-VL / Gemma receive labeled crops plus an optional full-frame overview, with the caption injected into the user prompt.

      Saves apply to the next case or reprocess. Analyses already in flight keep the config snapshot they started with. No app or Docker rebuild is required for ROI changes. RTSP URLs and other camera source settings are NOT controlled here — they live in config.yaml and the segment recorder must be restarted/reloaded for source URL changes to take effect.
